# CLAUDE.md

## プロジェクト概要

便利スクリプト集。既存の `shell/` および `perl/` にあるスクリプトを、以下の2つのターゲット向けに移植する。

- `unix/` : Linux/Mac向け、POSIX sh で記述
- `windows/` : Windows向け、PowerShell で記述

## ディレクトリ構成

```
.
├── shell/       # 移植元: シェルスクリプト
├── perl/        # 移植元: Perlスクリプト
├── unix/        # 移植先: POSIX sh スクリプト
└── windows/     # 移植先: PowerShell スクリプト
```

## 移植ルール

### 共通

- 各スクリプトは独立している（相互依存なし）
- 移植前に必ずファイルを読み込み、動作を完全に理解してから書き直すこと
- 機能・動作は元スクリプトと同等にすること
- スクリプト冒頭にコメントで元ファイル名と用途を記載すること

### unix/ (POSIX sh)

- shebang は `#!/bin/sh` を使用
- bash固有構文（`[[`, `(())`、配列など）は使わない
- 文字列処理は `sed` / `awk` / `cut` などの標準ツールを使う
- Perlスクリプトの移植も同様に POSIX sh で実装する

### windows/ (PowerShell)

- 拡張子は `.ps1`
- PowerShell 5.1 および 7.x 両対応を意識する
- Unix系コマンド（`grep`, `awk` 等）は使わず、PowerShell ネイティブのコマンドレットで実装する
- 文字コードは UTF-8 (BOM なし) を基本とする

## 作業上の注意

- 移植できない・判断が難しい箇所があれば、コメントに `TODO:` を残す
- 元スクリプトは変更しない（読み取り専用として扱う）

## プロジェクトの取り組み方について

* なるべくコードやコメントをシンプルに簡潔にすること。 
* コメントはUTF-8の日本語で書くこと。 
* 問題に取り組む前に、コードや仕様を読んで理解すること。
